# Roadmap

## Phase 0: Definition and validation

### Goal

Turn the concept into a buildable, testable product plan.

### Deliverables

- PRD baseline
- Scope freeze for MVP
- Core entity model
- Wireframes for Home, Today, Habits, Health, Finance, Review, and Quick Capture
- Scoring rubric draft
- Review templates for daily, weekly, and monthly cadence
- Authentication and security baseline
- MVP success-threshold baseline
- Privacy and data handling baseline

## Phase 1: MVP

### Goal

Deliver one reliable daily operating loop for planning, logging, scoring, and recurring review.

### Included scope

- Home
- Today
- Single owner login
- Habits and routines
- Health basics
- Expense tracking basics
- Goals, weekly priorities, and monthly focus
- Daily, weekly, and monthly reviews
- Quick capture
- Rule-based attention panel
- Rule-based scoring and momentum feedback
- In-app notifications

### Exit criteria

- Users can onboard into a useful default setup
- Users can complete morning, in-day, evening, weekly, and monthly workflows
- Common logging actions are fast enough for daily use
- Home provides a trustworthy status summary and understandable scoring feedback

## Phase 2: Connected life system

### Goal

Expand the product from a daily tracker into a more complete personal system.

### Candidate scope

- Life Admin module
- Savings goals and stronger finance summaries
- Goal, project, and milestone hierarchy
- Better scorecards and readiness indicators
- Deeper planning and routine templates
- Telegram integration
- Security hardening for broader internet exposure

## Phase 3: Smarter guidance and integrations

### Goal

Reduce manual effort and increase useful personalization.

### Candidate scope

- Calendar integration
- Financial data import or sync
- Health or wearable integrations
- Adaptive recommendations
- Better correlation insights
- Balance meter and richer life scoring

## Recommended sequencing principles

- Build the daily loop before deep analytics.
- Favor fewer modules with stronger utility over broad but shallow coverage.
- Add intelligence only after the underlying workflows are trusted and used consistently.
