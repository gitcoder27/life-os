# Product Vision

## Vision statement

Life OS is a personal command center that helps a user see their whole life in one place, decide what matters today, and stay consistent without feeling overwhelmed.

## Product promise

Run your life from one dashboard, build discipline through consistent daily action, and stay aligned across health, habits, money, and planning.

## The user

### Primary user

A busy independent professional who manages most of life alone and wants more structure without adding more complexity.

### Common traits

- Knowledge worker or system-oriented professional
- Feels friction from using too many disconnected apps
- Wants better consistency in health, money, routines, and long-term goals
- Prefers clarity, dashboards, and reliable systems over ad hoc memory

## Core jobs to be done

The product should help the user answer these questions every day:

- What matters today?
- What must I not forget?
- How am I doing physically and mentally?
- Am I staying on track overall?
- What should I do next?

## Product principles

### 1. One home, many life areas

The user should not need separate apps just to understand daily status across habits, health, money, and planning.

### 2. Action first

The interface should tell the user what needs attention and what to do next, not just show passive data.

### 3. Low-friction input

Logging must be quick enough to fit real life:

- one tap
- quick add
- templates
- defaults
- carry-forward behavior

### 4. Progressive depth

The dashboard should feel calm on first view and offer deeper detail only when the user asks for it.

### 5. Daily operation plus recurring review

The product must support quick daily use plus structured daily, weekly, and monthly reflection. Reflection is part of the product loop, not an optional add-on.

### 6. Balanced life, not just productivity

The system should reinforce health, money, admin, self-care, and future planning alongside tasks and output.

### 7. Transparent gamification

Scoring and momentum should help the user build discipline, but the feedback must stay understandable and tied to meaningful behavior.

## Emotional positioning

Life OS should make the user feel:

- organized
- in control
- less mentally overloaded
- healthier and more consistent
- more disciplined
- like life is moving forward

## Non-goals for MVP

- Team collaboration
- Household shared planning
- Deep accounting or banking integrations
- Advanced nutrition coaching
- Complex AI decision-making
- Replacing the user's external calendar or finance provider entirely

## Default product assumptions

- Responsive web app first
- Single-user product
- Simple login in MVP
- Manual input plus templates for MVP
- Private self-hosted deployment initially, with room for stronger security hardening later
- In-app notifications only in MVP
- Rule-based gamified scoring in MVP
- Tasks remain part of the daily operating flow, not a standalone module in the first release
- Limited smart summaries in MVP, not full automation
- Health, habit, expense, and review data are stored as private personal records
